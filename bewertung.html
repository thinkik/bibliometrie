Selbsttest, Kompetenznachweis und Abgaben

Dieser Lehrgang ist als Selbstlernangebot konzipiert. Die Übungen dienen der Selbstkontrolle, können aber auch in Workshops, Weiterbildungen oder internen Schulungen als Kompetenznachweis genutzt werden.

Übungslevels

Level 1: Verstehen

Level-1-Aufgaben prüfen zentrale Begriffe, Definitionen und typische Fehlinterpretationen. Sie eignen sich als kurze Wissenskontrolle nach einem Modul.

  • Begriff einer Definition zuordnen
  • korrekte Aussage auswählen
  • typische Fehlinterpretation erkennen

Level 2: Anwenden

Level-2-Aufgaben verlangen Anwendung, Berechnung oder kurze methodische Entscheidungen.

  • h-Index, JIF, NCS oder Kollaborationsraten berechnen
  • eine Datenquelle für einen Use-Case auswählen
  • eine kurze Methodikbox formulieren
  • Kennzahlen vorsichtig interpretieren

Level 3: Transfer

Level-3-Aufgaben sind kleine Fallstudien. Sie verlangen eine begründete Lösung für eine realistische Beratungssituation.

  • Methodik-Kasten für einen Report schreiben
  • Risiken einer Kennzahlenregel identifizieren
  • Dashboard-Konzept entwerfen
  • Responsible-Metrics-Leitlinien auf einen konkreten Fall anwenden

Optionales Portfolio

  1. Eine kurze Reflexion zu Responsible Metrics.
  2. Eine Methodikbox zu einer selbst gewählten Datenquelle.
  3. Eine Berechnung oder Interpretation einer Kennzahl.
  4. Eine kurze Fallanalyse zu einem Reporting- oder Beratungsszenario.
  5. Eine Liste der wichtigsten Limitationen und Datenqualitätsfragen.

Bewertungsvorschlag

KriteriumBeschreibungGewichtung
Fachliche KorrektheitBegriffe, Kennzahlen und Datenquellen werden korrekt verwendet.30 %
Methodische TransparenzQuelle, Zeitraum, Dokumenttypen, Counting, Stichtag und Limitationen werden genannt.30 %
Responsible-Metrics-ReflexionKennzahlen werden nicht mechanisch, sondern kontextualisiert genutzt.25 %
Verständliche KommunikationErgebnisse werden adressatengerecht und nachvollziehbar formuliert.15 %

Mindeststandard für gute Lösungen

Eine gute Lösung nennt nicht nur eine Zahl, sondern erklärt:

  • was die Zahl misst,
  • welche Datenquelle verwendet wurde,
  • welches Zeitfenster gilt,
  • welche Dokumenttypen einbezogen wurden,
  • welche Limitationen bestehen,
  • welche Schlussfolgerungen zulässig sind,
  • welche Schlussfolgerungen ausdrücklich nicht zulässig sind.
